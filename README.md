# AI-Attendance-System
# 点迹 APP - DianJi Check-in App
基于 Android Studio 开发的智能定位打卡应用。

## 项目简介
本项目是一款面向巡检、签到、位置管理的移动端打卡应用，通过 GPS 定位 + 地图可视化实现精准位置打卡，支持打卡点管理、距离判断、历史记录查询等功能。
项目完整覆盖需求分析、界面设计、功能开发、单元测试、UI 自动化测试与安全实践，是移动端开发与 AI 数据采集场景结合的典型实践。

## 核心功能
- 打卡点管理：添加、删除、修改、分类管理打卡地点（名称/经纬度/分类）
- 地图可视化：基于腾讯地图 SDK 展示打卡点分布与实时位置
- 智能打卡判断：300 米范围内自动允许打卡，保证打卡有效性
- 打卡记录：记录时间/地点，支持按成功/失败筛选查看
- 抽屉式菜单 + Fragment 架构：界面简洁、切换流畅、体验统一

## 技术栈
- 开发环境：Android Studio
- 架构：单 Activity + 多 Fragment
- 地图：腾讯地图 SDK（定位、地图展示、距离计算）
- 界面：DrawerLayout、NavigationView、RecyclerView
- 数据存储：本地数据持久化
- 测试：单元测试、Espresso 自动化 UI 测试
- 安全：权限最小化、位置权限动态申请、异常捕获、输入合法性校验

## 项目亮点
- 移动端作为 AI 系统的数据采集终端，可用于用户行为分析、位置预测、时序数据建模
- 完整的移动应用开发流程，具备 AI 模型落地到端侧设备的工程能力
- 位置感知、行为记录，是智能巡检、智慧管理系统的基础模块

## 开发与调试
- 支持真机 GPS 定位
- 支持模拟器模拟经纬度调试
- 经纬度合法性校验，避免地图 SDK 参数异常导致闪退
- RecyclerView 数据自动刷新，保证界面与数据同步

## 项目结构
- Hello04409Activity：主页面（框架管理）
- MapCheckFragment：地图打卡
- CheckPointManageFragment：打卡点管理
- CheckInHistoryFragment：打卡记录查询
